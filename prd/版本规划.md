# 个人学术主页 - 版本规划

## 产品概述

### 核心定位
一个极简、专业、易维护的个人学术主页，用于展示个人履历、科研成果、新闻动态及联系方式。

### 核心价值
- **极简主义**：去除冗余装饰，信息密度高，重点突出内容
- **易维护性**：通过修改数据文件快速更新内容
- **专业展示**：针对学术场景优化的布局和功能（News滚动框、纯文本论文列表等）

### 目标用户
- 教授、潜在合作者、企业招聘人员、同行研究者

## 版本规划路线图

**功能清单**：
1. 双栏响应式布局
2. 个人资料侧边栏
3. 顶部导航（锚点跳转）
4. About/News/Publications/Awards/Experience/Services七大板块

---

### V0.2 - UI/UX优化

**目标**：优化视觉体验，解决间距过大的问题

**功能清单**：
1. **优化垂直间距**
   - 减小section之间的margin-bottom（当前64px → 40-48px）
   - 减小sidebar与main-content之间的gap（当前64px → 32-40px）
   - 微调各组件内部间距
2. **优化排版密度**
   - 调整publication-item的padding
   - 优化news-item的间距
   - 统一各模块标题的样式
3. **视觉细节优化**
   - 添加hover状态的过渡效果
   - 优化滚动条样式
   - 添加页面加载动画（可选）
4. **移动端体验优化**
   - 调整移动端间距
   - 优化触摸交互

**不做什么**：功能变更、技术栈调整

**验证点**：
- 视觉上更紧凑、专业
- 各模块间距协调统一
- 移动端体验良好

**预计工作量**：0.5天

**影响文件**：
- `css/variables.css` - 调整间距变量
- `css/components.css` - 优化组件间距
- `css/layout.css` - 优化布局间距

---

### V0.3 - 在线编辑功能

**目标**：支持在页面上直接编辑内容并保存到本地

**功能清单**：
1. **编辑模式切换**
   - 添加"编辑模式"开关按钮
   - 编辑模式下显示可编辑区域指示器
2. **富文本编辑**
   - 集成轻量级富文本编辑器（如Quill.js或SimpleMDE）
   - 支持About Me的富文本编辑
   - 支持News/Research Interests的编辑
3. **表单编辑**
   - Publications的表单化编辑（添加/删除/修改）
   - Awards的表单化编辑
   - Experience的表单化编辑
4. **本地保存**
   - 下载修改后的data.js文件
   - 或者使用localStorage临时保存
5. **头像上传**
   - 支持拖拽上传头像
   - 自动裁剪/调整尺寸
   - 预览功能

**不做什么**：云端同步、多用户编辑、版本历史

**验证点**：
- 能在页面上编辑所有可编辑内容
- 编辑后能正确保存到本地文件
- 头像上传功能正常

**预计工作量**：1-2天

**技术选型**：
- 编辑器：Quill.js 或 TinyMCE（轻量级）
- 文件下载：Blob API
- 头像处理：cropper.js

**影响文件**：
- `index.html` - 添加编辑模式UI
- `js/editor.js` - 新增编辑器逻辑
- `js/data.js` - 数据结构保持不变
- `css/editor.css` - 新增编辑器样式

---

4. **数据校验**
   - 必填字段检查

**不做什么**：云端备份、协作编辑

**验证点**：
- 能正确导出和导入数据
- 版本快照功能正常

**影响文件**：

## 待确认的关键信息

### 技术选型确认
1. **富文本编辑器**：Quill.js vs TinyMCE vs 自研？
2. **头像处理**：是否需要裁剪功能？使用cropper.js还是简单缩放？
3. **数据存储方案**：
   - 纯前端方案：localStorage + 文件下载
   - 还是接入简单的后端（如Firebase/Supabase）？

1. **编辑功能范围**：是否只需要编辑文本内容，还是需要支持完整的内容管理（增删改查）？
2. **多语言需求**：中英文是否都需要完整支持，还是只需要中文？
3. **暗色模式**：是否必须包含在V1.0中？
4. **统计功能**：是否必须接入第三方统计，还是本地简单统计即可？

### 优先级确认
1. **间距优化**是否是最高优先级？是否需要先处理这个问题？
2. **在线编辑**是作为V0.3还是V0.5的核心功能？

## 核心原则

1. **渐进式交付**：每个版本都能独立使用，不依赖后续版本
2. **价值优先**：优先解决用户最痛的点（间距、编辑体验）
3. **快速验证**：每个版本都有明确的验证点
4. **明确边界**：每个版本明确"不做什么"
5. **可测量**：每个版本有清晰的完成标准
